# Example Reports

Two illustrative lint reports from a PM knowledge vault, included as style references for the skill and as samples for new users to see what the output looks like.

| File | What it illustrates |
|---|---|
| `example-broken-wikilinks-and-staleness.md` | A typical report focused on structural issues — broken wikilinks, oversized files, and bullets that have gone stale because no one ran `/archive-session` for two weeks |
| `example-cross-layer-drift.md` | A report dominated by semantic findings — verbatim duplication of facts across `MEMORY.md`, product `CLAUDE.md`, and multiple `_learnings.md` files |

Both reports lead with the most actionable finding, give specific file paths and line numbers, and end with a prioritized "Next Steps for User" list. Match this structure when writing new reports.

Project, ticket, and product names in these examples are illustrative placeholders. Substitute names from your own vault when you read these as a style reference.
